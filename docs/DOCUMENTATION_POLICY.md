# KaraCreate ドキュメント化方針・制約

## 目的

このプロジェクトの仕様を **手を抜かずに** ドキュメント化するための制約と方針を定める。
ドキュメントは「人間のための参照資料」であると同時に「AIエージェントが正確に実装・修正できる仕様書」として機能する。

---

## 制約（これを破ったらドキュメントとして不合格）

### C1: プレースホルダ禁止
- `TBD`、`TODO`、`後で書く`、`～など` のような曖昧な表現を使用しない
- 書けないなら「未実装」「未検証」として理由を明記する

### C2: コードとの整合性を検証可能にする
- 全てのドキュメント記述は、対応するソースファイルとパスを明記する
- 例: 「障子は PaperDoorBlock で実装されている」→ `content/paperDoor/PaperDoorBlock.java`
- ファイルパスは `src/main/java/mod/ckenja/karacreate/` を基準とした相対パスで記載

### C3: 数値・定数は実コードの値を転記
- ブロック強度、スタック数、速度範囲、処理時間などの数値は、コード上の実際の値を記載
- 「約〇〇」「だいたい〇〇」は禁止。正確な値を書く

### C4: 全要素を網羅する（省略禁止）
- ブロック・アイテム・レシピ・タグ・テクスチャ・Mixin —— 全要素を1つも漏らさず記載
- 「他にも同様のものがある」で逃げない。全て列挙する

### C5: 「何をするか」だけでなく「なぜそうしたか」を書く
- 設計判断の理由が推測できる場合は記載する
- Mixin が必要な理由、特定の BlockSetType を使う理由、等

### C6: 関連要素を相互参照する
- ブロックのドキュメントからレシピ、タグ、テクスチャ、レンダラーへのリンクを張る
- 孤立した情報を作らない

### C7: 既知の問題・不整合を隠さない
- 発見した問題点（例: mixins.json の重複エントリ等）はそのまま記載する

### C8: 言語は日本語を基本とする
- このModは日本文化テーマであり、開発者も日本語話者であるため
- 技術用語は英語のまま使用可（Block, Entity, Mixin, Recipe 等）

## ドキュメント構成

```
docs/
├── DOCUMENTATION_POLICY.md    ← 本ファイル（制約方針）
├── PROJECT_OVERVIEW.md        ← プロジェクト概要・ビルド・依存関係
├── ARCHITECTURE.md            ← パッケージ構成・設計思想・クラス関係
├── features/
│   ├── PAPER_DOORS.md         ← 障子・襖ドア（PaperDoor系）全仕様
│   ├── SMALL_DOORS.md         ← 半障子・半襖（SmallDoor系）全仕様
│   ├── MECHANICAL_COMPOSTER.md← メカニカルコンポスター全仕様
│   ├── BANNER_PATTERNS.md     ← 日本伝統文様バナーパターン全仕様
│   └── RECIPES.md             ← 全レシピ仕様
├── TAGS.md                    ← タグ体系
├── TEXTURES.md                ← テクスチャ一覧と用途
├── MIXINS.md                  ← Mixin一覧と変更内容
├── DATA_GENERATION.md         ← データジェネレーション仕様
├── LOCALIZATION.md            ← 多言語対応仕様
└── KNOWN_ISSUES.md            ← 既知の問題・不整合
```

## 各ドキュメントの必須セクション

### 機能仕様書（features/*）の必須項目
1. **概要**: 何であるか、ゲーム内でどう見えるか
2. **ブロック/アイテム一覧**: ID、プロパティ、強度、ツール等の全パラメータ
3. **振る舞い**: ゲーム内の具体的な動作（設置、操作、レッドストーン、Createとの連携）
4. **レンダリング**: 描画方法（標準/BlockEntityRenderer/Flywheel）
5. **レシピ**: クラフト方法の完全な記述
6. **タグ**: 所属するタグの一覧と理由
7. **テクスチャ**: 使用テクスチャファイルの一覧
8. **ソースファイル**: 関連する全Javaクラスのパスと役割
9. **既知の問題**: あれば記載
